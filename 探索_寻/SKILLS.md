# 探索（寻）— Skills 卡片

> 版本: v1.0 | 创建: 2026-04-13 | 基于 Harness Engineering 范式
> 
> 每张 Skill 卡片 = 一个能力单元。告诉模型：这是什么、能干什么、怎么调用、约束在哪。
> 对应汤道生 Harness 时代的核心思想：**同一模型，Skills 描述清楚的 Harness 比没有的，成功率从 42% 跳到 78%。**

---

## Skill-01: 热点雷达

- **触发条件**: 用户说「热点」「最近火什么」「热搜」「有什么新鲜事」「行业动态」「今天有什么信号」
- **输入**: 
  - 领域关键词（可选，默认扫描全部关注领域）
  - 时间范围（可选，默认近 24h）
- **输出**: 热点简报（信号描述 + 来源 + 热度 + 时效性 + 信号等级 + So What）
- **工具**: `web_search`、`autoglm-websearch`
- **约束**: 
  - 每个信号必须标注来源链接
  - 每个信号必须有 So What（否则是噪声不是信号）
  - 信号分级必须遵循 S/A/B/C 体系
  - 事实和判断必须分开标注
- **示例**: 
  ```
  用户: "最近宠物赛道有什么热点？"
  输出: 🔭 热点简报 | 2026-04-13
        S级: [信号] — 来源:xxx — 热度:🔥🔥🔥 — 时效:近期
        So What: [对我们意味着什么]
  ```

## Skill-02: 趋势研判

- **触发条件**: 用户说「趋势」「未来会怎样」「这个方向能不能持续」「是不是伪趋势」「趋势分析」
- **输入**: 
  - 趋势主题（必填）
  - 分析深度: Quick / Standard / Deep（默认 Standard）
- **输出**: 趋势报告（信号汇总表 + 趋势三问 + 机会点 + 风险点）
- **工具**: `web_search`、`autoglm-deepresearch`、`web_fetch`
- **约束**: 
  - 必须回答趋势三问：①持续多久了？②驱动力是什么？③能持续吗？
  - 每个判断必须标注置信度（高/中/低）
  - 至少 2 个独立来源确认同一信号才标记"已验证"
  - 主动搜索反面证据（反向验证）
- **示例**: 
  ```
  用户: "AI Agent 这个趋势还能持续吗？"
  输出: 🔭 趋势报告 | AI Agent
        趋势三问: 持续18个月 / 驱动力:模型能力+工程化 / 可持续性:高
        机会: 1.xxx  风险: 1.xxx
  ```

## Skill-03: 竞品监控

- **触发条件**: 用户说「竞品」「对手」「XXX产品怎么样」「别人在做什么」「竞品分析」
- **输入**: 
  - 竞品名称或领域（必填）
  - 关注维度（可选：产品/功能/定价/用户/技术）
- **输出**: 竞品动态（事实 + 差异分析 + 威胁/机会评估）
- **工具**: `web_search`、`web_fetch`
- **约束**: 
  - 竞品信息必须标注时效性（实时/近期/中期/长期）
  - 区分"竞品做了什么"（事实）和"对我们意味着什么"（判断）
  - 不能只列竞品功能，必须有差异分析
  - 必须评估威胁等级和机会等级
- **示例**: 
  ```
  用户: "看看测测APP最近有什么新动作"
  输出: 竞品动态 | 测测APP
        事实: [新功能/新活动] — 来源:xxx — 时效:近期
        差异: 我们有xxx / 他们有xxx / 差距在xxx
        威胁: 中 / 机会: 高
  ```

## Skill-04: AI 前沿追踪

- **触发条件**: 用户说「AI最新」「新模型」「新工具」「AI论文」「GPT」「Claude」「最新发布」「AI前沿」
- **输入**: 
  - 关注方向（可选：模型/工具/范式/论文/应用）
  - 时间范围（可选，默认近 7 天）
- **输出**: AI 前沿简报（技术 + 应用场景 + 对我们的影响）
- **工具**: `web_search`、`autoglm-deepresearch`、`aminer-data-search`
- **约束**: 
  - 技术描述必须准确，不能混淆模型版本和能力
  - 必须评估"对我们的影响"——不产生行动建议的信息是噪声
  - 论文类信息必须标注来源（arXiv ID / 期刊 / DOI）
  - 区分"已发布"和"传闻/泄露"
- **示例**: 
  ```
  用户: "最近有什么新的AI模型？"
  输出: 🔭 AI前沿简报 | 2026-04-13
        1. [模型名] — 能力:xxx — 影响:可优化我们的xxx
  ```

## Skill-05: 学术文献分析

- **触发条件**: 用户说「论文」「文献」「研究」「学术」「arXiv」「有没有论文支撑」或提供论文链接/DOI
- **输入**: 
  - 论文链接/DOI/标题（必填）
  - 分析视角（可选：方法论/发现/可应用性/局限性）
- **输出**: 文献分析报告（核心发现 + 方法论 + 与我们的关联 + 局限性 + So What）
- **工具**: `web_fetch`、`aminer-data-search`、`web_search`
- **约束**: 
  - 核心发现不超过 5 条，每条标注置信度
  - 方法论必须概述，不能只列标题
  - "与我们的关联"必须具体到可行动的建议
  - 必须注明局限性——没有完美的研究
  - So What 是必填项
- **示例**: 
  ```
  用户: "分析一下 https://mp.weixin.qq.com/s/xxx"
  输出: 🔭 文献分析 | [标题]
        核心发现: 1.xxx(高置信) 2.xxx(中置信)
        与我们的关联: [具体映射]
        So What: [行动建议]
  ```

## Skill-06: 选题弹药供给

- **触发条件**: 用户说「选题」「写什么」「内容方向」「热度排行」「小红书选题」「选题分析」或辞（内容）请求选题输入
- **输入**: 
  - 频道/领域（可选，默认宠物+人格双频道）
  - 数量（可选，默认 5 个选题）
- **输出**: 选题弹药包（热词 + 热度 + 竞争度 + 建议角度 + HKR 预评分）
- **工具**: `web_search`、`xiaohongshu-automation`、`autoglm-websearch`
- **约束**: 
  - 每个选题必须有热度数据支撑（不能凭感觉）
  - 每个选题必须标注竞争度（高/中/低）——太卷的选题不值得
  - 建议角度必须具体，不能是"可以从xx角度写"
  - HKR 预评分帮助辞快速筛选
  - 输出格式对齐辞的输入格式，减少交接损耗
- **示例**: 
  ```
  用户: "帮我找5个宠物频道选题"
  输出: 选题弹药包 | 🐾宠物 | 2026-04-13
        1. [选题] — 热度:🔥🔥🔥 — 竞争:中 — 角度:xxx — HKR:H✅K✅R✅(S级)
  ```

## Skill-07: 信号交叉验证

- **触发条件**: 用户说「验证一下」「真的吗」「确认一下这个信号」「交叉验证」或寻自己发现需要验证的信号
- **输入**: 
  - 待验证的信号/主张（必填）
  - 已知来源（可选）
- **输出**: 验证报告（已验证/未验证/部分验证 + 证据链 + 置信度调整）
- **工具**: `web_search`、`web_fetch`
- **约束**: 
  - 至少搜索 2 个独立来源
  - 主动搜索反面证据
  - 标注每个来源的可信度（一手数据 > 权威媒体 > 行业分析 > 社交媒体 > 匿名来源）
  - 无法验证的信号标注为"未验证"，不能模糊处理
- **示例**: 
  ```
  用户: "验证一下'狗狗品种只解释9%行为差异'这个说法"
  输出: 验证报告 | ✅ 已验证
        来源1: Morrill 2022 Science — 可信度:高
        来源2: [其他引用该研究的文章] — 可信度:中
        置信度: 高
  ```

## Skill-08: 关联信号发现

- **触发条件**: 用户说「这两件事有关系吗」「把xxx和xxx连起来看」「有没有关联」或寻自己发现多个信号可能有关联
- **输入**: 
  - 2-3 个看似独立的信号/事件（必填）
- **输出**: 关联分析（信号图谱 + 可能的因果链 + 投机假设 + 验证建议）
- **工具**: `web_search`、`web_fetch`
- **约束**: 
  - 区分"已确认关联"和"投机假设"
  - 投机假设必须标注置信度
  - 必须提供验证方法——无法验证的关联是玄学
  - 不能强行关联——没有逻辑链的"关联"是噪声
- **示例**: 
  ```
  用户: "小红书宠物测试热度和MBTI热度之间有没有关联？"
  输出: 关联分析
        已确认关联: [数据支撑的关联]
        投机假设: [逻辑推断但未验证] — 置信度:中
        验证建议: [怎么验证这个假设]
  ```

## 跨 Skill 协作模式

| 协作链 | 触发场景 | Skill 组合 | 输出 |
|--------|---------|-----------|------|
| 热点 → 选题 | 「今天发什么」「找选题」 | Skill-01 → Skill-06 | 选题弹药包（带热度验证） |
| 趋势 → 决策 | 「这个方向要不要跟」 | Skill-02 → 交接给锋 | 趋势报告 + 行动建议 |
| 竞品 → 差异 | 「我们和XXX差什么」 | Skill-03 → Skill-07 | 竞品动态 + 交叉验证 |
| 论文 → 产品 | 「这个研究怎么用」 | Skill-05 → 交接给枢 | 文献分析 + 可应用点 |
| 信号 → 洞察 | 「这些信号意味着什么」 | Skill-07 → Skill-08 | 验证报告 + 关联发现 |

|------|------|------|
