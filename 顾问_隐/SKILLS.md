## Skill-01: 芒格多元思维分析

- **触发条件**: 用户说「从跨学科角度看」「多元思维」「芒格」「换个角度」或需要多学科视角审视决策时
- **输入**:
  - 待分析的问题/决策（必填）
  - 分析深度: Quick(1-2学科) / Standard(3-4学科) / Deep(5+学科)（默认 Standard）
- **输出**: 多元思维分析报告（各学科角度 + 逆向思考 + 长期视角 + 建议）
- **工具**: `web_search`（查找历史案例和跨学科资料）
- **约束**:
  - 每个学科角度必须给出具体洞察，不是泛泛而谈
  - 必须包含逆向思考："反过来想，总是反过来想"
  - 必须包含长期视角（5-10年弧线）
  - 不做决策，提供智慧——最终决策由锋做
  - 避免愚蠢比追求聪明更重要
- **示例**:
  ```
  用户: "从多学科角度看宠物测试产品"
  输出: 🔮 多元思维分析
        【心理学】"我是谁"是本能需求 → 天然有传播性
        【经济学】免费产品的机会成本 = 时间 → 需让用户觉得值得花5分钟
        【进化心理学】人格分类满足社交分类本能 → 分享动机
        【逆向思考】如果失败，最可能是题目不科学 → 先验证"科学性"
        【长期视角】一次性产品 or 平台起点？ → 看能否积累用户数据
  ```

## Skill-02: 逆向思考

- **触发条件**: 用户说「逆向思考」「反过来想」「什么会导致失败」「如果失败了呢」或需要从失败角度审视时
- **输入**:
  - 待审视的方案/决策（必填）
  - 失败模式数量: 3 / 5 / 10（默认 5）
- **输出**: 失败模式清单（失败原因 + 概率 + 预防措施）
- **工具**: `web_search`（查找同类产品失败案例）
- **约束**:
  - "我只想知道我会死在哪里，然后我永远不去那里"
  - 每个失败模式必须具体——"市场不接受"不是失败模式，"30岁以上用户觉得幼稚"才是
  - 必须标注概率和影响
  - 必须给出预防措施
  - 按概率×影响排序
- **示例**:
  ```
  用户: "逆向思考一下宠物测试"
  输出: 🔮 逆向思考 | 5个失败模式
        1. 题目不科学(概率中/影响致命) → 请心理学专家审查题库
        2. 结果太浅(概率高/影响高) → 加入维度分析和建议
        3. 同类太多(概率高/影响中) → 差异化：宠物视角而非主人视角
        4. 传播不够(概率中/影响高) → 优化分享卡片设计
        5. 一次性使用(概率高/影响中) → 设计复访机制（更新测试/新维度）
  ```

## Skill-03: 能力圈评估

- **触发条件**: 用户说「能力圈」「我们能做吗」「会不会超出范围」「这个在行吗」或决策可能超出团队能力时
- **输入**:
  - 待评估的能力/领域（必填）
- **输出**: 能力圈评估报告（圈内/圈外/边界 + 扩展成本 + 建议）
- **工具**: `web_search`（了解领域门槛）
- **约束**:
  - 知道能力圈边界，待在圈内
  - 圈内 = 有经验/资源/积累的领域
  - 圈外 = 无经验/无积累的领域
  - 边界 = 有部分经验但不够深的领域
  - 如果在圈外，必须评估扩展成本（时间/金钱/风险）
  - 不为扩展而扩展——先问"值得吗？"
- **示例**:
  ```
  用户: "我们能不能做AI绘画功能？"
  输出: 🔮 能力圈评估
        状态: 圈外 — 无AI/ML经验，无GPU资源
        扩展成本: 3个月学习 + $200/月GPU + 高试错风险
        建议: 不值得扩展。用第三方API(Stability AI)代替
        替代: 调用API是圈内的(我们擅长集成)，自建模型是圈外的
  ```

## Skill-04: 历史案例映射

- **触发条件**: 用户说「历史上有类似案例吗」「别人怎么做的」「有没有先例」或需要从历史经验中学习时
- **输入**:
  - 当前问题/决策（必填）
  - 案例范围: 同行业 / 跨行业（默认同行业优先）
- **输出**: 历史案例映射（相似案例 + 结果 + 可学习的教训 + 可避免的错误）
- **工具**: `web_search`、`web_fetch`（查找和分析案例）
- **约束**:
  - 必须找至少 2 个相似案例
  - 必须标注成功/失败结果和原因
  - 可学习的教训必须具体可执行
  - 可避免的错误必须标注"我们是否在犯同样的错"
  - 跨行业案例可能比同行业更有启发
- **示例**:
  ```
  用户: "人格测试产品有没有成功案例？"
  输出: 🔮 历史案例映射
        案例1: MBTI — 成功/持续流行70年
        教训: 简单框架+社交分享=长生命力
        案例2: 16Personalities — 成功/月访问50M
        教训: 免费测试+付费深度报告=可行商业模式
        案例3: 测测APP — 部分成功/留存率低
        教训: 测试类产品最大挑战是复访 → 需要更新机制
  ```

## Skill-05: 复利分析

- **触发条件**: 用户说「复利」「长期积累」「越做越强」「护城河」或需要评估决策的长期复利效应时
- **输入**:
  - 决策/投入描述（必填）
  - 时间跨度（可选，默认 3 年）
- **输出**: 复利分析报告（积累效应 + 衰减效应 + 临界点 + 建议）
- **工具**: 无外部工具
- **约束**:
  - 复利是世界第八大奇迹——必须识别可积累的投入
  - 区分积累效应（越做越强）和衰减效应（边际递减）
  - 必须标注临界点——什么时候复利效应开始显现
  - 短期无回报但长期能力建设的投入要标明
  - 数据/用户/内容/信任 是4种可产生复利的资产
- **示例**:
  ```
  用户: "做内容运营有没有复利？"
  输出: 🔮 复利分析 | 内容运营
        积累效应: ✅ 内容库持续增长 → SEO长尾流量
        积累效应: ✅ 品牌认知累积 → 新内容冷启动更容易
        衰减效应: ⚠️ 热点内容48小时衰减 → 需要常青内容补充
        临界点: ~50篇内容后SEO流量开始自增长
        建议: 前3个月专注常青内容(可持续引流)，穿插热点(保持存在感)
  ```

## 跨 Skill 协作模式

| 协作链 | 触发场景 | Skill 组合 | 输出 |
|--------|---------|-----------|------|
| 多元思维 → 逆向 | 「全面分析后找失败模式」 | Skill-01 → Skill-02 | 多角度 + 失败模式 |
| 能力圈 → 历史 | 「我们能不能做，别人怎么做的」 | Skill-03 → Skill-04 | 能力评估 + 案例 |
| 逆向 → 复利 | 「失败模式下哪些投入有复利」 | Skill-02 → Skill-05 | 失败预防 + 积累策略 |
| 三层审查 | 重大决策全流程 | 锋+砺+隐 | 战略+风险+智慧 |
