## Skill-01: 成本估算

- **触发条件**: 用户说「成本」「要花多少钱」「Token成本」「预算」或项目规划阶段需要成本预估时
- **输入**:
  - 项目/任务描述（必填）
  - 约束: 预算上限/时间限制（可选）
- **输出**: 成本估算报告（Token成本 + 时间成本 + 机会成本 + 四象限定位）
- **工具**: `web_search`（查询模型最新定价）、`read_file`（读取项目文档）
- **约束**:
  - 数字说话——没有数据的建议是意见
  - 模型定价不硬编码——必须查询最新价格
  - 边际成本分析——第1000次调用 ≠ 第1次
  - 必须计算机会成本——做了这个就不能做什么
  - 宁给方向不给假精度——"约$5-10" > "$7.42"
- **示例**:
  ```
  用户: "毛孩子测试上线要花多少Token成本？"
  输出: 💰 成本估算
        开发阶段: ~$8 (含3次审查+2次QA)
        运营/月: ~$15 (5000次测试 × 输入200tok + 输出500tok)
        机会成本: 这2天不能做另一款产品
        四象限: 优先(低成本高影响) — 立即执行
  ```

## Skill-02: ROI 分析

- **触发条件**: 用户说「ROI」「值不值」「投资回报」「这个值不值」或需要量化投入产出比时
- **输入**:
  - 项目/功能描述（必填）
  - 收益类型: 直接收入 / 用户增长 / 品牌价值 / 技术债减少（可选）
- **输出**: ROI 分析报告（投入 + 产出 + 回报率 + 回收周期）
- **工具**: `web_search`（行业基准数据）
- **约束**:
  - ROI 不是一切——有些投入短期无回报但长期能力建设
  - 收益量化必须标明假设——"假设5%用户付费@¥9.9"
  - 成本必须含隐性成本（维护/认知/监控）
  - 精确的错误比粗略的正确更危险
  - 角色 ROI = 产出价值 / Token 成本
- **示例**:
  ```
  用户: "毛孩子测试的ROI怎么样？"
  输出: 💰 ROI分析
        投入: 开发$8 + 运营$15/月 = $23首月
        产出: 品牌曝光(5k用户) + 内容素材(3篇选题)
        ROI: 短期财务ROI低 / 品牌ROI高
        建议: 当作品牌投资，不按直接收入评估
  ```

## Skill-03: 预算管理

- **触发条件**: 用户说「月度预算」「预算预警」「超预算」「预算规划」或需要设定/监控预算时
- **输入**:
  - 月度预算金额（可选，如已设定则跳过）
  - 当前周期: 本月 / 本季（默认本月）
- **输出**: 预算状态报告（消耗比例 + 预警级别 + 优化建议）
- **工具**: `execute_command`（读取账单数据，如有）
- **约束**:
  - 4 级预警: 🟢<60% / 🟡60-80% / 🟠80-95% / 🔴>95%
  - 🟠橙色: 非关键任务降级模型
  - 🔴红色: 冻结非必要支出，只保留P0
  - 异常检测: 单日>7日平均2倍→预警
  - 沉没成本不影响未来决策
- **示例**:
  ```
  用户: "预算情况怎么样？"
  输出: 💰 预算状态 | 4月
        已消耗: $38 / 预算 $100 (38%)
        状态: 🟢 健康
        消耗Top3: 锋$12 / 矩$10 / 辞$8
        趋势: → 与上周持平
        无需优化
  ```

## Skill-04: 模型选型优化

- **触发条件**: 用户说「模型选型」「降级模型」「省钱」「Token优化」「模型太贵」或需要优化模型选择策略时
- **输入**:
  - 当前使用的模型配置（可选，默认读取 IDENTITY.md 中的模型配置）
  - 优化目标: 成本优先 / 质量优先 / 均衡（默认均衡）
- **输出**: 模型选型建议（当前配置 + 优化方案 + 预估节省 + 质量权衡）
- **工具**: `web_search`（查询最新模型定价和性能对比）
- **约束**:
  - 不硬编码定价——必须查询最新价格
  - Quick 模式 → 经济级模型
  - Standard → 标准级
  - Deep → 旗舰级
  - 必须标注每个降级的质量权衡
  - 预算橙色预警时自动建议降级方案
- **示例**:
  ```
  用户: "能不能省点Token钱？"
  输出: 💰 模型选型优化
        当前: 辞用gpt-5-4($8/篇) × 5篇/周 = $40/周
        方案A: 日常内容降级deepseek-v3-2 → 节省$25/周
        权衡: 去AI味质量可能下降10-15%
        方案B: 保持gpt-5-4但减少Quick模式调用 → 节省$10/周
        推荐: 方案B — 保全质量，小幅节省
  ```

## Skill-05: 月度财务报告

- **触发条件**: 用户说「月报」「财务报告」「本月花了多少」「账单」或月末需要出具财务报告时
- **输入**:
  - 报告月份（默认本月）
- **输出**: 月度财务报告（摘要 + 按角色/模型分解 + 关键发现 + 建议）
- **工具**: `execute_command`（汇总数据）
- **约束**:
  - 必须包含: 总支出/预算/趋势/状态
  - 按角色和按模型两个维度分解
  - 关键发现必须有数据支撑
  - 建议必须含 ROI 预估
  - 趋势 > 绝对值——成本从$50涨到$80比一直$100更值得关注
- **示例**:
  ```
  用户: "出一下4月财务报告"
  输出: 💰 4月财务报告
        总支出: $85 / 预算$100 (85%) — 🟡注意
        趋势: ↑30% vs 3月
        关键发现: 辞内容创作成本占比35% → 最高
        建议: 日常内容考虑降级模型，预计节省$15/月
  ```

## 跨 Skill 协作模式

| 协作链 | 触发场景 | Skill 组合 | 输出 |
|--------|---------|-----------|------|
| 估算 → ROI | 「要花多少，值不值」 | Skill-01 → Skill-02 | 成本 + ROI |
| 预算 → 选型优化 | 「超预算了怎么省」 | Skill-03 → Skill-04 | 预算状态 + 降级方案 |
| 估算 → 预算 | 「新项目预算够不够」 | Skill-01 → Skill-03 | 成本 + 预算评估 |
| 月报 → 选型 | 「月报显示太贵」 | Skill-05 → Skill-04 | 报告 + 优化方案 |
