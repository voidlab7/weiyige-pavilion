# 贡献指南

感谢你对维弈阁的关注！在贡献之前，请仔细阅读以下条款。

## 贡献者协议

### 1. 贡献即授权

当你向本项目提交 Pull Request、Issue、代码、文档或任何形式的贡献时，即表示你：

- 同意你的贡献以 **GPL-3.0** 协议授权给本项目
- 同意原作者（voidlab7）作为项目维护者，有权在 GPL-3.0 框架下管理、合并、修改你的贡献
- 确认你的贡献是你自己的原创工作，或你有权以 GPL-3.0 授权提交

### 2. 署名权

- 所有贡献者都会在 git 历史中保留署名
- 贡献者保留其贡献部分的署名权（GPL-3.0 第 5 条）
- 项目维护者有权调整代码结构、合并贡献，但不会删除贡献者的 git 署名

### 3. 衍生作品

如果你 fork 本项目并创建衍生作品：

- **必须**以 GPL-3.0 或兼容协议发布
- **必须**保留对 gstack（Garry Tan）的署名
- **必须**保留对维弈阁（voidlab7）的署名
- **必须**在衍生项目的显著位置注明来源和变更
- **不能**将本项目内容以闭源或更严格协议重新发布

### 4. 不可声明为"第一作者"

- 本项目的原创设计者和第一作者是 **voidlab7**
- gstack 的原创设计者是 **Garry Tan**
- 贡献者是贡献者，不是项目的"第一作者"或"联合创始人"
- 在任何宣传、文章、演讲中引用本项目时，不得暗示你是项目的创始者

### 5. 商业使用

GPL-3.0 允许商业使用，但必须：

- 开源你的衍生作品（相同协议）
- 保留所有署名
- 不得将本项目注册为商标或申请专利

## 如何贡献

1. Fork 本仓库
2. 创建功能分支（`git checkout -b feature/your-feature`）
3. 提交修改（`git commit -m 'Add some feature'`）
4. 推送分支（`git push origin feature/your-feature`）
5. 创建 Pull Request

### 贡献范围

欢迎贡献：

- 新的角色技能（skills）
- 角色人设优化（IDENTITY / SOUL）
- 安装脚本改进
- 文档和翻译
- Bug 修复
