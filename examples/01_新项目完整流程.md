# 示例 01：新项目完整流程（Team 模式）

> 适用于 `@启` / `@team` / `@auto` 触发的全链路自动编排。

## 用户输入

```text
@team auto 我有个新产品想法：做一个毛孩子人格测试 V2。请从想法评估、需求、设计、开发、测试、安全审计走完整流程。
```

## 路由判断

- 触发词：`@team`、`auto`、`完整流程`
- 路由到：`启·执事`
- 运行方式：team 模式
- 编排模式：auto

## 团队创建

主 Agent 读取 `执事_启/start.md`，创建团队：

```text
team_create(team_name="weiyige-pet-personality-v2")
```

按链路并行 spawn 成员，`name` 必须使用角色单字：

| subagent_name | name | 初始状态 |
|--------------|------|---------|
| 启·执事 | 启 | 立即工作 |
| 砺·合伙人 | 砺 | 待命 |
| 锋·CEO | 锋 | 待命 |
| 枢·PM | 枢 | 待命 |
| 矩·架构 | 矩 | 待命 |
| 绘·设计 | 绘 | 待命 |
| 铸·开发 | 铸 | 待命 |
| 鉴·QA | 鉴 | 待命 |
| 盾·安全 | 盾 | 待命 |

## 标准链路

```text
启
 ├─ 砺：Office Hours v2，验证想法是否值得做
 ├─ 隐：Deep 模式下独立二评（如需要）
 ├─ 绘：UI 类想法低保真视觉草图（如需要）
 ├─ 锋：方向审批
 ├─ 枢：PRD
 ├─ 矩 + 绘：工程审查 + 设计审查
 ├─ 铸：代码实现 + 左移检查
 ├─ 鉴：QA 测试
 ├─ 盾：安全审计
 └─ 启：汇总报告 + 关闭团队
```

## 产物目录

```text
ai-workspace/{task_id}/
├── state.json
├── progress-board.md
└── artifacts/
    ├── 01-ideation/     # 砺 Office Hours 设计文档
    ├── 02-requirement/  # 枢 PRD
    ├── 03-design/       # 矩/绘审查报告和草图
    ├── 04-development/  # 铸实现记录和左移检查
    ├── 05-testing/      # 鉴/盾测试与安全报告
    └── 06-summary/      # 启汇总报告
```

## 每个成员的交接块

必须对齐 `PROTOCOL.md §2.2`：

```markdown
---
## 📤 交接块（Handoff）

- **来源**: [Agent 名称]
- **阶段**: [当前阶段]
- **产出类型**: [PRD / 设计文档 / 审查报告 / 测试报告]
- **产物文件**: [实际路径]
- **状态**: [通过 / 有条件通过 / 未通过 / 需要信息]
- **关键决策**:
  1. [决策]: [结论]
- **开放问题**:
  1. [问题]
- **下游建议**: [建议交谁做什么]
- **阻塞项**: [无 / 阻塞说明]
---
```

## 停顿条件

auto 模式下默认不停顿，只有以下情况暂停：

- `未通过` 且迭代已达上限；
- 需要用户补充信息；
- 涉及破坏性操作或发布动作；
- 上游产物文件不可读；
- team 工具不可用。

## 完成标准

- `06-summary/` 有启的汇总报告；
- 所有阶段产物路径可读；
- 所有成员已 `shutdown_request`；
- `team_delete()` 完成；
- 未解决问题写入 `TODO.md` 或项目 `STATUS.md`。
