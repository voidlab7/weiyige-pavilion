# 异常容错矩阵

> 核心原则：核心流程必须成功；辅助功能失败不阻塞。

## 硬/软依赖

| 依赖 | 类型 | 失败后果 |
|------|------|---------|
| 产物文件落盘 | **硬** | 阻塞下游，必须解决 |
| state.json 更新 | **硬** | 丢失进度，必须补写 |
| Agent spawn/唤醒 | **硬** | 流程断裂，暂停报告 |
| git commit/push | **软** | 跳过，后续补做 |
| 评分 Skill 加载 | **软** | 降级为人工判断 |
| 进度看板更新 | **软** | 跳过，不阻塞 |
| runtime-knowledge 写入 | **软** | 跳过，不影响流程 |

## 异常处理预案

| 异常 | 类型 | 处理策略 |
|------|------|---------|
| Agent spawn 失败 | 硬 | 重试 1 次 → 失败暂停，报告用户 |
| 产物文件不存在 | 硬 | 打回上游 Agent，要求补写（计入 retry_count） |
| 打回 ≥3 次 | 门禁 | 标记 human-needed，可选跳到下一阶段 |
| state.json 损坏 | 恢复 | 从 ai-workspace 产物目录反推状态 → 备份 → 人工确认 |
| 用户中途退出 | 正常 | state.json 已记录，下次自动恢复 |
| Agent 超时无响应 | 硬 | 60s 后重发 send_message → 仍无响应标记 failed |
| Skill 加载失败 | 软 | 降级为 Leader 内置审查能力 |
| 文件系统异常 | 硬 | 产物内嵌交接块（标注降级），门禁标记 ⚠️ 有条件通过 |
