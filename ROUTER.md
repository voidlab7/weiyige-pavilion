# 维弈阁路由器（Weiyige Router）

> 版本: v1.0 | 创建: 2026-04-11 | 状态: 生效中

用户不应该需要记住 10 个 Agent 的名字和触发条件。路由器根据**用户意图**自动分发到正确的 Agent。

---

## 一、路由架构

```
用户输入（自然语言）
       │
       ▼
┌──────────────┐
│  意图识别层    │  ← Step 1: 判断用户想做什么
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  层级路由层    │  ← Step 2: 判断属于哪个层
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  Agent 选择层  │  ← Step 3: 在层内选择具体 Agent
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  模式选择层    │  ← Step 4: Quick / Standard / Deep
└──────────────┘
```

## 三、路由决策树

```
用户输入
  │
  ├─ 包含"想法/方向/要不要做/战略" ──→ 战略层
  │    ├─ 需要多角度深度分析 ──→ 隐
  │    ├─ 需要质疑/挑战/风险 ──→ 砺
  │    └─ 需要最终决策/审查  ──→ 锋
  │
  ├─ 包含"需求/PRD/计划/排期/文案/内容" ──→ 执行层
  │    ├─ 产品需求相关 ──→ 枢
  │    └─ 内容创作相关 ──→ 辞
  │
  ├─ 包含"架构/设计/代码/技术/UI" ──→ 质量层（构建）
  │    ├─ 技术架构/代码审查相关 ──→ 矩
  │    ├─ 代码实现/开发相关   ──→ 铸
  │    └─ UI/UX/视觉相关     ──→ 绘
  │
  ├─ 包含"测试/Bug/安全/漏洞" ──→ 质量层（验证）
  │    ├─ 功能测试/Bug 相关 ──→ 鉴
  │    └─ 安全/威胁相关     ──→ 盾
  │
  ├─ 包含"成本/预算/Token/ROI" ──→ 运营层
  │    └─ 财务相关 ──→ 算
  │
  └─ 意图不明确 ──→ 追问用户
       "你想要：
        A) 评估方向/做决策 → 战略层
        B) 定义需求/写内容 → 执行层
        C) 技术设计/代码审查 → 质量层
        D) 测试/安全检查 → 质量层
        E) 成本/预算分析 → 运营层"
```

## 四、复合意图处理

当用户输入包含多个意图时，按以下规则处理：

### 4.1 串行处理（有依赖关系）

```
"我有个想法，帮我评估一下，然后写 PRD"
  → 先路由到 砺（质疑想法）
  → 通过后路由到 锋（方向确认）
  → 确认后路由到 枢（写 PRD）
```

### 4.2 并行处理（无依赖关系）

```
"帮我做安全审计，同时看看成本"
  → 并行路由到 盾（安全审计）+ 算（成本分析）
```

### 4.3 流程触发（完整流程）

```
"启动新项目"
  → 按 PROTOCOL.md 标准协作链路，从构思阶段开始
  → 自动按阶段依次激活 Agent
```

## 五、基于项目阶段的智能路由

路由器会参考 STATUS.md 中的当前阶段，优先路由到当前阶段的主导 Agent：

| 当前阶段 | 默认路由 | 说明 |
|---------|---------|------|
| 构思 | 锋 → 隐 → 砺 | 战略层主导 |
| 需求定义 | 枢 | 执行层主导 |
| 设计 | 绘 + 矩 | 质量层主导 |
| 开发 | 铸（代码实现） | 质量层主导 |
| 测试 | 鉴 + 盾 | 质量层主导 |
| 发布 | 枢（协调） | 执行层主导 |
| 运营 | 算 + 辞 | 运营层主导 |

**覆盖规则**：用户可以在任何阶段直接指定 Agent，路由器不阻拦。

## 六、模式自动选择

路由器根据任务规模自动建议模式：

| 信号 | 建议模式 | 说明 |
|------|---------|------|
| "快速看一下"、"简单问个" | Quick | 2 分钟内完成 |
| 无特殊标记 | Standard | 默认模式 |
| "深度"、"全面"、"彻底"、"大版本" | Deep | 完整流程 |
| 任务预计 < 30 分钟 | Quick | 自动降级 |
| 任务涉及 > 5 个文件 | Deep | 自动升级 |

## 八、路由失败处理

### 8.1 无法判断意图

```
🤔 我不确定你需要哪个角色来帮忙。你的需求更接近：

A) 🎯 战略决策 — 方向、计划、要不要做 → @锋 / @隐 / @砺
B) 📋 需求规划 — PRD、需求拆解、排期 → @枢
C) ✍️ 内容创作 — 文案、公众号、去 AI 味 → @辞
D) 🏗️ 技术设计 — 架构、代码审查 → @矩
E) ⚒️ 代码实现 — 写代码、开发、修 Bug → @铸
F) 🎨 UI/UX — 界面设计、设计审查 → @绘
G) 🧪 测试 — 找 Bug、质量检查 → @鉴
H) 🛡️ 安全 — 安全审计、威胁建模 → @盾
I) 💰 财务 — 成本、预算、ROI → @算
J) 🔭 探索 — 热点、趋势、竞品调研 → @寻

请选择，或者更详细地描述你的需求。
```

### 8.2 多个 Agent 都可能适用

当多个 Agent 都可能适用时，路由器展示选项并给出推荐：

```
📌 你的需求可能涉及多个角色：

推荐路径：先 @砺（质疑需求真实性）→ 再 @枢（写 PRD）
替代路径：直接 @枢（跳过质疑，直接写 PRD）

选择哪个？
```
