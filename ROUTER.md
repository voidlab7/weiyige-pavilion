# 维弈阁路由器（Weiyige Router）

> 版本: v1.3 | 创建: 2026-04-11 | 更新: 2026-05-03 | 状态: 生效中

用户不应该需要记住 13 个 Agent 的名字和触发条件。路由器根据**用户意图**自动分发到正确的 Agent。

---

## 一、路由架构

```text
用户输入（自然语言）
       │
       ▼
┌──────────────┐
│  意图识别层    │  ← Step 1: 判断用户想做什么
└──────┬───────┘
       ▼
┌──────────────┐
│  层级路由层    │  ← Step 2: 判断属于哪个层
└──────┬───────┘
       ▼
┌──────────────┐
│  Agent 选择层  │  ← Step 3: 在层内选择具体 Agent
└──────┬───────┘
       ▼
┌──────────────┐
│  模式选择层    │  ← Step 4: Quick / Standard / Deep / Team
└──────────────┘
```

## 三、关键边界

| 问题 | 找谁 | 不找谁 |
|------|------|--------|
| 一个想法值不值得继续 | 砺 | 枢（不要直接写 PRD） |
| 方向最终拍板 | 锋 | 砺 / 隐（只建议，不审批） |
| 需要独立二评 | 隐 | 砺（砺是质疑，不是二评） |
| 多角色自动串联 | 启 | 主会话手工假装 team |
| 写代码 / 修 Bug | 铸 | 矩（矩审方案，不亲自实现） |
| UI 草图 / 设计审查 | 绘 | 砺（砺只给设计约束） |
| 市场 / 热点 / 竞品探索 | 寻 | 枢（枢负责需求，不负责广域搜索） |

## 五、复合意图处理

### 5.1 串行处理（有依赖）

```text
“我有个想法，帮我评估一下，然后写 PRD”
  → 砺（Office Hours）
  → 锋（方向确认）
  → 枢（PRD）
```

### 5.2 并行处理（无依赖）

```text
“帮我做安全审计，同时看看成本”
  → 并行路由到 盾 + 算
```

### 5.3 完整流程

```text
“启动新项目 / 自动跑完 / 全链路”
  → 启按 PROTOCOL.md 和 执事_启/start.md 编排
```

## 六、基于项目阶段的智能路由

| 当前阶段 | 默认路由 | 说明 |
|---------|---------|------|
| 构思 | 砺 → 隐 → 锋 | 先质疑和二评，再拍板 |
| 需求定义 | 枢 | 执行层主导 |
| 设计 | 绘 + 矩 | 设计与工程并行审查 |
| 开发 | 铸 | 代码实现和左移检查 |
| 测试 | 鉴 + 盾 | 功能质量 + 安全 |
| 发布 | 枢 | 协调发布 |
| 运营 | 算 + 辞 | 成本与内容 |

**覆盖规则**：用户可以在任何阶段直接指定 Agent，路由器不阻拦。

## 七、模式自动选择

| 信号 | 建议模式 | 说明 |
|------|---------|------|
| “快速看一下”“简单问个” | Quick | 2-5 分钟内完成 |
| 无特殊标记 | Standard | 默认模式 |
| “深度”“全面”“彻底”“大版本” | Deep | 完整流程 |
| “自动跑完”“全链路”“走完流程” | Team / 启调度 | 多角色串联 |
| 任务涉及 > 5 个文件 | Deep | 自动升级 |

## 八、运行方式选择

| 方式 | 典型口令 | 是否启动 task | 是否阻塞当前主会话 | 适用场景 |
|------|----------|--------------|-------------------|----------|
| **新会话直连** | `@寻-live` / `直接扮演寻，不要 task` | 否 | 否 | 单角色长任务 |
| **当前会话 inline** | `@寻-inline` / `直接以寻身份回答` | 否 | 是 | 快速角色化回答 |
| **启调度** | `@启` / `@team` / `启动团队流程` | 由启决定 | 可能阻塞或走 team mode | 多角色串联 |

规则：

1. 用户明确说“不想在主会话开 task / 不要阻塞主会话”时，建议新会话直连。
2. 用户明确说“不要 task / 不要 subagent / 直接扮演某角色”时，进入 inline / 直连模式，不等待 Leader。
3. 用户需要多角色调度、全链路、自动跑完时，才交给 `@启`。

## 十、路由失败处理

```text
我不确定你需要哪个角色。你的需求更接近：
A) 战略决策 — 方向、计划、要不要做 → @锋 / @隐 / @砺
B) 需求规划 — PRD、需求拆解、排期 → @枢
C) 内容创作 — 文案、公众号、去 AI 味 → @辞
D) 信息探索 — 热点、趋势、竞品、论文 → @寻
E) 技术设计 — 架构、工程审查 → @矩
F) 代码实现 — 写代码、开发、修 Bug → @铸
G) UI/UX — 界面设计、视觉草图、设计审查 → @绘
H) 测试 — 找 Bug、质量检查 → @鉴
I) 安全 — 安全审计、威胁建模 → @盾
J) 财务 — 成本、预算、ROI → @算
K) 自动编排 — 多角色串联、全链路 → @启 / @team
```
