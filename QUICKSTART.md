# 维弈阁快速入门（Quickstart）

> 5 分钟上手，10 分钟完成第一次完整的 Agent 调用。

---

## 一、维弈阁是什么？

维弈阁是一个**多角色 AI Agent 团队**。你是唯一的决策者，10 个 AI Agent 是你的专业团队：

| 快捷指令 | Agent | 一句话职责 |
|---------|-------|-----------|
| `@锋` / `@ceo` | CEO_锋 | 方向对不对？做不做？ |
| `@枢` / `@pm` | PM_枢 | 需求拆解、写 PRD |
| `@矩` / `@arch` | 架构_矩 | 技术方案、代码审查 |
| `@绘` / `@design` | 设计_绘 | UI/UX 设计、设计审查 |
| `@鉴` / `@qa` | QA_鉴 | 找 Bug、质量测试 |
| `@盾` / `@sec` | 安全_盾 | 安全审计、威胁建模 |
| `@算` / `@cfo` | 财务_算 | 成本分析、预算管理 |
| `@辞` / `@content` | 内容_辞 | 文案创作、去 AI 味 |
| `@隐` / `@advisor` | 顾问_隐 | 多角度深度分析 |
| `@砺` / `@devil` | 合伙人_砺 | 质疑挑战、魔鬼代言人 |

**不知道该找谁？** 直接描述你的需求，路由器会自动帮你找到对的人（见 `ROUTER.md`）。

## 二、3 分钟上手

### Step 1: 了解三档模式

每个 Agent 都支持三档模式，按任务大小选择：

| 模式 | 适用场景 | 耗时 |
|------|---------|------|
| **Quick** | 小问题、快速确认 | < 2 分钟 |
| **Standard** | 日常任务（默认） | 5-30 分钟 |
| **Deep** | 重大决策、大版本 | 30 分钟+ |

使用方式：
```
@quick @锋 这个方向对吗？          ← Quick 模式
@锋 审查一下这个计划               ← Standard 模式（默认）
@deep @锋 全面审查这个战略方案       ← Deep 模式
```

### Step 2: 试一下

**场景 A：你有个新想法**
```
我有个想法：做一个 AI 驱动的宠物健康管理 App。帮我评估一下。
```
→ 路由器会自动分发到 砺（质疑想法）→ 锋（方向确认）

**场景 B：你要写需求**
```
@枢 帮我写一个用户登录功能的 PRD
```
→ 直接调用 PM_枢

**场景 C：你要审查代码**
```
@quick @矩 看看这个 API 设计有没有问题
```
→ Quick 模式调用架构_矩

### Step 3: 看交接块

每个 Agent 完成工作后，会输出一个**交接块**，告诉你下一步该找谁：

```
📤 交接块（Handoff）
- 来源: PM_枢
- 状态: 通过
- 下游建议: 建议交 @矩 做架构评审
```

按建议继续调用下一个 Agent 即可。

## 五、最佳实践

### ✅ 推荐做法

1. **小任务用 Quick 模式** — 不是每件事都需要走全流程
2. **跟着交接块走** — Agent 会告诉你下一步该找谁
3. **重大决策走完整流程** — 锋→隐→砺→枢→矩，一个都别跳
4. **让 Agent 积累记忆** — 每次会话结束后，Agent 会自动沉淀经验
5. **用快捷指令** — `@锋`、`@枢` 比描述意图更快

### ❌ 避免的做法

1. **不要跳过质疑环节** — 砺的质疑能帮你避免 80% 的错误决策
2. **不要同时给太多 Agent 发任务** — 一次聚焦一个阶段
3. **不要忽略交接块** — 那是 Agent 之间信息传递的关键
4. **不要手动编辑 memory 文件** — 让 Agent 自己管理

## 六、FAQ

**Q: 我只有一个小问题，需要走完整流程吗？**
A: 不需要。用 `@quick @[Agent]` 即可，2 分钟搞定。

**Q: Agent 之间会自动协作吗？**
A: Agent 会在交接块中建议下一步，但需要你来触发。你是调度者。

**Q: 我可以同时用多个 Agent 吗？**
A: 可以，但建议按阶段顺序来。并行适合无依赖的任务（如安全审计 + 成本分析）。

**Q: Agent 的记忆会一直保留吗？**
A: 会。记忆分三层：短期（当前任务）、中期（当前项目）、长期（团队知识库）。详见 `MEMORY.md`。

**Q: 我可以自定义 Agent 吗？**
A: 可以。Fork 后修改 `IDENTITY.md` 和 `SOUL.md` 即可。
