# 🚨 审核流程执行指令（非提醒，是指令）

> 借鉴: "你的下一个动作只能是 X"
>
> **本文件是审核动作的唯一入口。审查类 Agent 在输出审查结论前必须读取本文件。**

---

## 执行序列（严格按顺序，不可调换、不可跳过）

```
Step 1: 读取本文件（你刚完成）
Step 2: 读取 rules/review-scoring.md（评分规范）
Step 3: 按评分维度逐项审查产物，输出评分报告
Step 4: 读取对应 gates/gate-*.md，逐条勾选门禁检查项
Step 5: 综合评分报告 + 门禁检查 → 输出最终裁定
```

## ⚠️ 关键设计

- **评分在门禁之前**：先看产物质量（Step 3），再看流程完成度（Step 4）
- **评分不可跳过**：没有评分报告的审查结论 = 无效审查 = 不得流转
- **门禁不可跳过**：没有门禁检查的审查结论 = 无效审查 = 不得流转

## 合法审核结论的必要条件

你的审核结论**必须同时包含以下两部分**才被视为有效：

1. ✅ `review-scoring.md` 格式的多维度评分报告（含总分、各维度分数、问题列表）
2. ✅ `gates/gate-*.md` 逐条检查结果

缺少任何一部分 = **审核无效** = 不得流转到下一阶段。
