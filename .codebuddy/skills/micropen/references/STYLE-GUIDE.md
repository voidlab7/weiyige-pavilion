# STYLE-GUIDE.md — 写作风格硬规范

> 每次写作前必读。规范 = 硬标准，不是建议。

---

## 一、开头模式（3 选 1，不可违反）

### 模式 A：反直觉暴击（优先使用）
第一句话就打破读者预期。

```
✅ 我花了一天时间，让 AI 从"看不见图片"变成"会自己交叉验证"
✅ 加 Agent 是最廉价的勤奋
✅ 维弈阁从 12 个 Agent 升级到 13 个，最大的变化不是加了一个"铸·开发"
```

**禁止**：
- ❌ "今天和大家分享..."
- ❌ "最近我在想一个问题..."
- ❌ "相信很多人都有这样的困惑..."

### 模式 B：数字冲击
用具体数字开场，引发好奇。

```
✅ 15 个专业角色，全部通过 Markdown 文件驱动
✅ 6 天 5 万 Star，YC 总裁开源的东西到底有什么用
✅ 一个月跑了 300 条链路，我总结出 3 个多 Agent 协作的真相
```

### 模式 C：场景切入
直接把读者拉到一个具体场景。

```
✅ 周六晚上 11 点，我对着 MiniMax 说："帮我识别这张图里的表格"
✅ 上周我让 Claude 写了一个安装脚本，跑完它给我报了三个错
```

---

## 二、第二句必须"炸"

第一句拉住注意力，**第二句必须给出"冲击性结论/数据/观点"**，否则读者划走。

```
例子：
第一句：我花了一天时间，让 AI 从"看不见图片"变成"会自己交叉验证"。
第二句：过程中我发现一个反常识的事——大模型的"多模态能力"，一半是营销。
         ↑ 这句就是"炸"
```

**检查：**
读第二句，如果没有让读者产生"啊？"或"认真的吗？"的感觉，重写。

## 三、标题公式

### 公式 1：数字 + 成果 + 反差（最常用）
```
✅ 《从 12 到 13 个 Agent，我终于想明白了这件事》
✅ 《6 天 5 万 Star：Y Combinator 总裁是怎么把 Claude Code 变成虚拟团队的》
✅ 《一天时间，一个 AI 从"瞎子"变"火眼金睛"》
```

### 公式 2：反直觉断言
```
✅ 《多 Agent 协作的真相：加 Agent 不难，难的是链路跑通》
✅ 《加 Agent 是最廉价的勤奋》
✅ 《大多数人都错了：AI 不是越多越强》
```

### 公式 3：独家视角
```
✅ 《我用 Hermes 的 7 层架构，给自己的 AI 团队做了一次体检》
✅ 《逆向 Copilot 那 3 天，我终于看懂了 AI 编程的本质》
```

### 标题禁用词
- ❌ "浅谈"、"初探"、"分享一下"、"一点思考"
- ❌ "必看"、"干货"、"实战教程"（LOW）
- ❌ "震惊"、"重磅"、"太强了"（标题党）

### 标题自检
- [ ] 有具体数字吗？
- [ ] 有反差 / 反直觉吗？
- [ ] 读者看完标题能预判收益吗？
- [ ] 不是"浅谈 XXX"这种式？

---

## 四、结尾要求：弹射 ≠ 降落

### ❌ 降落式结尾（禁止）
```
总的来说...
希望对大家有帮助...
大家有什么想法欢迎交流...
我们下次再见~
```

### ✅ 弹射式结尾（要求）

**方式 1：金句收官**
```
编排层和执行层分离，才是多 Agent 协作的核心。
```

**方式 2：行动钩子**
```
这五条过不了，先别急着加 Agent。
```

**方式 3：反问抛出**
```
造更多 Agent 是最廉价的勤奋——真正难的事，你想过怎么解吗？
```

**方式 4：信息召回**
```
microlab.top 第一个产品已经上线。
下一个实验，可能是你的。
```

**检查：** 读者读完结尾，脑子里是否留下一句话？如果没有，重写。

## 五、文章骨架（公众号 Standard）

```
【标题】数字 + 成果 + 反差
  ↓
【副标题/导语】1 句话核心观点
  ↓
【开头】反直觉 / 数字 / 场景（3 选 1）
  ↓ 第二句必炸
  ↓
【H2：一个反直觉的事实】
  建立主论点，1-2 段
  ↓
【H2：背景/问题/为什么难】
  引入背景，说明问题严重性
  ↓
【H2：正文主体】× 3-5 个小节
  每个 H2 = 一个论点
  每段最多 3-4 行（公众号阅读习惯）
  ↓
【H2：把一切收束到主论点】
  升华，但不降落
  ↓
【H2：金句段】
  > 一句话可以被引用的金句
  ↓
【弹射结尾】
  钩子/行动/反问
  ↓
【签名区】
  - Microlab 实验日志
  - microlab.top
  - 关联：小红书「Microlab趣味实验」
```

## 六、段落节奏规则

### 短段为王
- 公众号阅读：手机竖屏，每段 1-3 行最佳
- 超过 4 行 → 拆段
- 禁止：一口气写 200 字一段

### 长短交替
```
❌ 3 段都是长段（读者疲劳）
❌ 3 段都是短段（节奏碎）
✅ 长 → 短 → 长 → 短 → 一句话成段（强调）
```

### 一句话成段（高级技巧）
```
...（上段解释）

**编排层和执行层分离，才是关键。**  ← 单独成段，加粗

...（下段承接）
```

## 七、H2 小标题规范

### ✅ 用"观点/结论"当标题
```
✅ 加 Agent 是最廉价的勤奋
✅ 启·执事的本质：编排层，不是执行层
✅ 真正重要的不是 Agent 数量
```

### ❌ 禁止用"话题/名词"当标题
```
❌ 关于多 Agent 协作
❌ 启·执事介绍
❌ Agent 的数量问题
```

## 八、表格 / 列表使用规范

### 表格适合：
- 对比（A vs B）
- 矩阵（维度 × 对象）
- 清单（检查项/步骤）

### 列表适合：
- 3-5 个并列要点
- 步骤流程
- 对比项

### 禁止：
- ❌ 把正文全写成列表（没人愿意看 bullet 堆砌）
- ❌ 表格超过 5 列 7 行（手机看不清）

---

## 九、代码块使用规范

### 适合放代码的场景：
- 展示核心逻辑（不超过 10 行）
- 命令行操作
- 配置示例

### 不适合：
- 超过 30 行的代码（放 GitHub 链接）
- 伪代码（改用流程文字描述）

## 十、自检清单（交付前必过）

### 开头
- [ ] 第一句符合 A/B/C 三种模式之一？
- [ ] 第二句"炸"了吗？
- [ ] 没有"大家好"、"今天分享"、"最近在想"？

### 标题
- [ ] 3 个备选，带热度评分？
- [ ] 最优选项符合公式 1/2/3 之一？
- [ ] 没有禁用词？

### 结尾
- [ ] 是弹射，不是降落？
- [ ] 读完留下一句可记忆的话？

### 节奏
- [ ] 段落 ≤ 4 行？
- [ ] 长短段交替？
- [ ] 至少有 1 处"一句话成段"？

### 观点
- [ ] 至少 1 个反直觉观点？
- [ ] 每个抽象观点都有具体例子/数字/代码？
- [ ] 没有空洞的"赋能"、"助力"、"数字化转型"？

### 完整性
- [ ] 有 H2 小标题且是"观点/结论"式？
- [ ] 有至少 1 个表格或对比清单？
- [ ] 有签名区（品牌信息+链接）？

---

## 违反规范的后果

这不是"建议"，是"铁律"。
任何一条不通过 → 交付作废 → 重写。
