# 技术方案文档审核标准

## 审核维度与权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 架构设计 | 25% | 整体架构合理性、模块划分、分层设计 |
| 技术选型 | 20% | 技术栈选择合理性、与现有系统兼容性 |
| 详细设计 | 20% | 接口设计、数据模型、核心流程 |
| 可扩展性 | 15% | 水平扩展能力、未来演进考虑 |
| 安全与可靠性 | 10% | 安全设计、容错机制、监控告警 |
| 文档质量 | 10% | 表达清晰、图表规范、完整性 |

## 各维度详细检查项

### 架构设计（25%）

- [ ] **整体架构图**：有清晰的系统架构图（组件图/部署图）
- [ ] **模块划分**：各模块职责清晰，边界明确，符合单一职责原则
- [ ] **分层合理**：分层设计合理（如：接入层/业务层/数据层），层间依赖方向正确
- [ ] **接口定义**：模块间接口定义清晰，避免紧耦合
- [ ] **数据流向**：核心数据流向有说明（数据流图或时序图）
- [ ] **与现有系统集成**：与现有系统的集成方式已说明

**扣分参考**：
- 无架构图：-5~-8分
- 模块职责不清晰：-3~-5分
- 存在明显的循环依赖：-3~-5分
- 缺少数据流说明：-2~-3分

### 技术选型（20%）

- [ ] **选型理由**：每个关键技术选型有明确的理由
- [ ] **对比分析**：对主要技术方案有对比分析（优缺点）
- [ ] **团队熟悉度**：考虑了团队技术栈熟悉度
- [ ] **生态成熟度**：选用的技术有足够的社区支持和稳定性
- [ ] **许可证合规**：开源组件的许可证与业务需求兼容
- [ ] **版本兼容性**：与现有系统的版本依赖无冲突

**扣分参考**：
- 技术选型无理由：-3~-5分
- 引入不必要的复杂技术：-2~-4分
- 存在明显更优方案未考虑：-2~-3分

### 详细设计（20%）

- [ ] **API设计**：接口定义完整（请求/响应格式、错误码、认证方式）
- [ ] **数据模型**：核心数据结构/表结构设计合理，字段含义清晰
- [ ] **核心算法**：关键业务逻辑/算法有伪代码或流程说明
- [ ] **状态机**：有状态流转的场景有状态机图
- [ ] **并发处理**：并发场景（竞态条件、锁机制）有说明
- [ ] **缓存策略**：缓存使用场景、失效策略有说明

**扣分参考**：
- API设计不完整：-3~-5分
- 数据模型设计有明显缺陷：-3~-5分
- 并发问题未考虑：-3~-5分
- 缺少关键流程说明：-2~-3分

### 可扩展性（15%）

- [ ] **水平扩展**：支持水平扩展（无状态设计或有状态说明）
- [ ] **性能目标**：有明确的性能指标（QPS、延迟、并发数）
- [ ] **容量规划**：数据量增长的应对方案（分库分表、归档等）
- [ ] **功能扩展点**：预留了合理的功能扩展点，避免过度设计
- [ ] **降级方案**：关键功能有降级/熔断策略

**扣分参考**：
- 无法水平扩展的单点设计：-5~-8分
- 无性能目标：-2~-3分
- 无降级方案：-2~-3分

### 安全与可靠性（10%）

- [ ] **认证鉴权**：接口有合适的认证鉴权机制
- [ ] **数据安全**：敏感数据加密存储/传输
- [ ] **输入校验**：有防注入、XSS等安全防护说明
- [ ] **容错机制**：超时、重试、熔断等容错策略
- [ ] **数据一致性**：分布式场景下的一致性保证方案
- [ ] **监控告警**：关键指标的监控和告警方案

**扣分参考**：
- 无认证鉴权：-3~-5分
- 敏感数据明文存储：-3~-5分
- 无容错机制：-2~-3分

### 文档质量（10%）

- [ ] **结构完整**：包含背景、目标、方案设计、部署方案等章节
- [ ] **图表清晰**：架构图、流程图等图表清晰可读
- [ ] **术语一致**：技术术语使用一致，必要时有说明
- [ ] **版本记录**：有版本号和评审记录
- [ ] **可操作性**：开发人员可据此直接实现，无需大量额外沟通

## 常见问题示例

### 🔴 严重问题
- 核心模块无详细设计，开发无法据此实现
- 存在单点故障且无容错方案
- 安全设计明显缺失（如无鉴权的对外接口）
- 数据一致性问题未处理

### 🟡 中等问题
- 技术选型缺乏充分论证
- 接口设计不完整（缺少错误码定义）
- 性能目标不明确
- 缺少监控方案

### 🟢 建议改进
- 补充更详细的容量规划
- 增加压测方案说明
- 补充运维部署文档
- 考虑灰度发布策略
