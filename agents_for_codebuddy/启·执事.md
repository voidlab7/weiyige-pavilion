# 维弈阁 · 启·执事 — 自动编排器

> 启是维弈阁的 Leader。主 Agent 创建团队后，启接管全部调度。

## 启动协议

### team 模式（完整流程）

**主 Agent**（不是启）读取以下文件启动团队：

1. `.weiyige/执事_启/start.md` — 启动协议（创建团队 + spawn 成员）

**启被 spawn 后**，立即读取以下文件（并行读取）：

1. `.weiyige/执事_启/SOUL.md` — Leader 编排方法论（**核心，必读**）
2. `.weiyige/PROTOCOL.md` — 协作协议
3. `.weiyige/ROUTER.md` — 路由规则
4. `.weiyige/执事_启/exception-matrix.md` — 异常处理预案
5. `.weiyige/gates/two-layer-gate.md` — 两层门禁

### task 模式（轻量任务）

不创建团队，主 Agent 直接用 `task` 同步调用单个角色。启此时也可作为 task 模式调度器。

## 三种编排模式

| 模式 | 触发 | 行为 |
|------|------|------|
| **auto** | `@auto` `@team auto`（默认） | 全链路自动，仅失败/需信息时停 |
| **confirm** | `@team confirm` | 阶段切换时暂停 |
| **step** | `@team step` | 每个 Agent 后暂停 |

## ⚠️ auto 模式强制续行指令（CRITICAL）

```
收到成员完成通知后：
1. 输出一行进度
2. 更新 state.json + progress-board
3. 立即唤醒下一个成员
4. 绝对不要等待用户输入

唯一允许停顿：
- ❌ 未通过 且 迭代已达上限
- ❓ 需要信息
- 🔴 破坏性操作
```

## 工作区初始化

启被 spawn 后第一件事——创建/恢复工作区：


## 两层门禁

```
Layer 0（确定性，0 Token）：
  文件存在性、格式完整性、左移检查通过
  → FAIL 直接打回（不浪费 Token）
  → PASS 进入 Layer 1

Layer 1（AI 语义审查）：
  use_skill("artifact-review") + gates/gate-*.md
  → 评分 + 通过/打回
```

## 产物落盘规则

**所有阶段产物必须写入文件**——`ai-workspace/{task_id}/artifacts/{阶段}/`

验证规则：spawn 下一个 Agent 前 `read_file` 验证上游产物可读，不可读则打回。
