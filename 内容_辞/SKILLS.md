## Skill-01: 内容创作

- **触发条件**: 用户说「写文章」「写文案」「帮我写」「创作内容」或需要产出内容时
- **输入**:
  - 主题/选题（必填）
  - 目标平台: 小红书 / 公众号 / 抖音 / 通用（必填）
  - 创作深度: Quick / Standard / Deep（默认 Standard）
- **输出**: 平台终稿（经过 De-AI-ify + Humanizer + 平台适配）
- **工具**: `web_search`（选题调研）、`read_file`（读取 skill 文件）
- **约束**:
  - 所有内容必须经过 De-AI-ify + Humanizer（Skill-02/03），未经处理的是半成品
  - 洞察优先——没有洞察的内容是噪音
  - 3 秒法则——前 3 秒抓不住就没用
  - 标题是承诺——正文必须兑现
  - 具体性法则——"3小时" > "很长时间"
- **示例**:
  ```
  用户: "写一篇小红书，养猫后才知道的事"
  输出: ✍️ 内容创作 | 小红书 | Standard
        选题卡: 洞察="猫不能说话是最大的孤独" 情绪=共鸣+心疼
        Phase 3 AI初稿 → Phase 4 De-AI-ify → Phase 5 Humanizer → Phase 6 平台适配
        终稿: [经过去AI味+人味处理的完整文案]
  ```

## Skill-02: De-AI-ify

- **触发条件**: 用户说「去AI味」「De-AI」「太AI了」「润色一下」或 AI 初稿需要去味时
- **输入**:
  - 待处理文本（必填）
- **输出**: 去味稿（AI 常用词替换 + 结构打破）
- **工具**: `read_file`（读取 `skills/de-ai-ify.md`）
- **约束**:
  - 必删词表: 非常→挺/真、各种→具体说、致力于→删、赋能→帮、助力→帮...
  - 打破"首先…其次…最后…"的工整结构
  - 减少"的"字
  - 长短句交错，故意不齐整
  - 关键点独立成段
- **示例**:
  ```
  输入: "我们致力于为用户提供全方位的服务，助力宠物主人更好地了解自己的宠物"
  输出: "我们帮养宠人搞懂自家毛孩子到底在想什么"
  ```

## Skill-03: Humanizer

- **触发条件**: 用户说「加人味」「Humanize」「太正式」「像人写的」或去味稿需要加人味时
- **输入**:
  - 待处理文本（必填，通常是 De-AI-ify 后的稿子）
- **输出**: 人味稿（口语化 + 真实感受 + 具体细节 + 停顿留白）
- **工具**: `read_file`（读取 `skills/humanizer.md`）
- **约束**:
  - 加入思考过程: "怎么说呢"、"老实说"、"坦白讲"
  - 加入真实感受: "头疼"、"真香"、"心累"
  - 加入具体细节: "3个小时"、"半年"、"上周三"
  - 加入停顿: 段落间空行、省略号
  - 不能过度——人味不是灌水
- **示例**:
  ```
  输入: "养猫后才发现猫的独立性很强"
  输出: "养猫之前，我以为猫不需要人。怎么说呢……直到有天加班到凌晨，回家发现它就蹲在门口。那一刻我才意识到，独立不等于不需要。"
  ```

## Skill-04: 选题策划

- **触发条件**: 用户说「选题」「写什么」「内容方向」「选题分析」或需要规划内容方向时
- **输入**:
  - 频道/领域（可选，默认宠物+人格双频道）
  - 时间范围: 本周 / 本月（默认本周）
- **输出**: 选题卡列表（主题 + 洞察 + 情绪 + 平台 + 预期指标）
- **工具**: `web_search`（热点扫描）、`xiaohongshu-automation`（小红书热榜）
- **约束**:
  - 必须用三层选题法: 热点(L1) / 常青(L2) / 品牌独占(L3)
  - 优先级: L3 > L2 > L1
  - 每个选题必须有洞察——"宠物很重要"不是洞察
  - 数据驱动——不靠灵感靠数据
  - 每个选题必须标注目标情绪和目标平台
- **示例**:
  ```
  用户: "帮我找5个选题"
  输出: ✍️ 选题策划 | 本周
        L3-01: "毛孩子不会说话，但它的测试结果会" — 品牌:独占 — 情绪:共鸣
        L2-02: "养猫后才知道的10件事" — 常青 — 情绪:认同
        L1-03: "猫咪性格测试火了" — 热点(<48h) — 情绪:好奇
  ```

## Skill-05: 爆款复盘

- **触发条件**: 用户说「复盘」「为什么爆/扑」「分析一下数据」「内容复盘」或内容发布后 48 小时需要分析时
- **输入**:
  - 内容标题/链接（必填）
  - 数据: 阅读/点赞/收藏/转发/评论（可选，如有）
- **输出**: 复盘报告（数据分析 + 模式提炼 + 行动项）
- **工具**: `web_search`（对比同类内容数据）
- **约束**:
  - 必须分析: 选题/洞察/情绪/标题/平台 5 个维度
  - 必须提炼可复制的模式
  - 必须给出具体行动项——不是"下次注意"
  - 收藏率 > 点赞率 = 内容有实用价值
  - 转发率 > 点赞率 = 内容有社交货币价值
- **示例**:
  ```
  用户: "复盘一下上篇小红书"
  输出: ✍️ 爆款复盘
        数据: 阅读5k / 点赞300 / 收藏800 / 转发150
        分析: 收藏>点赞 → 实用价值高 / 转发偏低 → 社交货币不足
        模式: 实用型选题 + 情绪型标题 = 高收藏低传播
        行动: 下次加入"分享给同样养猫的朋友"引导 → 提升传播
  ```

## Skill-06: 平台适配

- **触发条件**: 用户说「适配小红书」「改成公众号版本」「抖音脚本」或同一内容需要多平台分发时
- **输入**:
  - 源内容（必填）
  - 目标平台: 小红书 / 公众号 / 抖音（必填）
- **输出**: 平台终稿（适配目标平台的格式/节奏/情绪）
- **工具**: `read_file`（读取平台特性速查表）
- **约束**:
  - 小红书: emoji + 悬念标题 + 碎片化分段 → 收藏率>点赞率
  - 公众号: 金句 + 逻辑链 + 深度 → 完读率+转发率
  - 抖音: hook + 节奏 + 反转 → 完播率+评论率
  - 同一洞察，不同讲法——不是简单改格式
  - 标题适配: 小红书悬念+数字+emoji / 公众号价值承诺 / 抖音封面文案=标题
- **示例**:
  ```
  用户: "把这篇适配成小红书版本"
  输出: ✍️ 平台适配 | 小红书
        标题: 😱测完才知道…我家猫竟然是这种人格？！
        开头: 前3行决定命运 → 悬念hook
        结构: emoji分段 + 短句 + 留白
        CTA: 收藏→下次给猫测 / 评论→你家猫是什么人格
  ```

## 跨 Skill 协作模式

| 协作链 | 触发场景 | Skill 组合 | 输出 |
|--------|---------|-----------|------|
| 选题 → 创作 | 「帮我选题然后写」 | Skill-04 → Skill-01 | 选题卡 + 内容终稿 |
| 创作 → De-AI → Human | 「写一篇文章」 | Skill-01 → 02 → 03 | 完整内容流水线 |
| 创作 → 平台适配 | 「写完适配多平台」 | Skill-01 → Skill-06 | 多平台终稿 |
| 发布 → 复盘 | 「复盘这篇内容」 | Skill-05 → Skill-04 | 复盘 + 下期选题优化 |
| 选题（寻→辞） | 寻供给选题弹药 | 寻 Skill-06 → 辞 Skill-04 | 选题弹药包 → 选题卡 |
