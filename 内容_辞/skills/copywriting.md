# 文案撰写技能

## 概述

文案不是"写字"——是**用最少的字传递最强的信息**。好文案 = 精准洞察 × 情绪共鸣 × 行动驱动。

---

## 文案类型速查

| 类型 | 目标 | 字数 | 核心要求 |
|------|------|------|---------|
| **品牌 Slogan** | 记住品牌 | 3-10 字 | 独特、好记、有态度 |
| **产品标题** | 点击/打开 | 10-30 字 | 悬念、利益、具体 |
| **功能描述** | 理解价值 | 20-50 字 | 用户视角、场景化 |
| **CTA 按钮** | 立即行动 | 2-6 字 | 动词开头、低门槛 |
| **错误提示** | 不慌张 | 10-30 字 | 说人话、给出路 |
| **空状态** | 不迷茫 | 10-30 字 | 引导行动、不冷冰冰 |
| **推送通知** | 打开 App | 15-30 字 | 紧迫感、个性化 |
| **社交分享** | 被转发 | 20-50 字 | 社交货币、情绪价值 |

## 文案四步法

### Step 1: 明确目标

```
这段文案要让用户做什么？
  □ 记住（品牌认知）
  □ 点击（引导行动）
  □ 理解（传递信息）
  □ 分享（社交传播）
  □ 购买（转化成交）
```

### Step 2: 找到洞察

```
用户在这个场景下：
  - 在想什么？（心理状态）
  - 在怕什么？（痛点/焦虑）
  - 在期待什么？（渴望/目标）
  - 会被什么打动？（情绪触发点）
```

### Step 3: 写初稿（3 个版本）

每次至少写 3 个版本：
- **版本 A**: 理性诉求（讲道理、摆数据）
- **版本 B**: 感性诉求（讲故事、触情绪）
- **版本 C**: 行动诉求（制造紧迫、降低门槛）

### Step 4: 打磨（删删删）

```
检查清单：
  □ 能不能再短一半？
  □ 有没有废话？（"的"字太多？修饰词太多？）
  □ 换成口语说出来自然吗？
  □ 用户 3 秒能理解吗？
  □ 经过 De-AI-ify 处理了吗？
```

## 场景模板

### 品牌 Slogan

**公式**: [态度/动作] + [品牌独特性]

```
好例子：
  "Just Do It" — 态度 + 行动
  "Think Different" — 态度 + 差异化
  "让天下没有难做的生意" — 承诺 + 具体场景

坏例子：
  "致力于为用户提供优质服务" — 空洞、无差异化
  "引领行业创新" — 谁都能说
```

### 产品标题（小红书/公众号）

**公式**: [数字/悬念] + [具体利益] + [情绪钩子]

```
好例子：
  "3 个月瘦 20 斤，我只做了一件事"
  "被裁后第 47 天，我收到了 3 个 offer"
  "养猫 5 年才知道的 8 个真相"

坏例子：
  "如何减肥成功" — 太泛、无悬念
  "分享一些养猫经验" — 无具体性、无情绪
```

### 功能描述

**公式**: [用户场景] + [产品动作] + [具体结果]

```
好例子：
  "拍照识花 — 对准花朵拍一张，1 秒告诉你它叫什么"
  "智能翻译 — 对着菜单拍一下，外语秒变中文"

坏例子：
  "强大的图像识别功能" — 技术视角，不是用户视角
  "全方位的翻译解决方案" — AI 味、空洞
```

### CTA 按钮

**公式**: [动词] + [具体结果]（可选：+ [低门槛暗示]）

```
好例子：
  "免费试用 7 天"（动词 + 结果 + 低门槛）
  "立即查看结果"（动词 + 具体结果）
  "一键生成"（低门槛 + 动词）

坏例子：
  "提交"（无吸引力）
  "了解更多"（模糊、无动力）
  "点击这里"（废话）
```

### 错误提示

**公式**: [发生了什么] + [为什么] + [怎么办]

```
好例子：
  "网络开小差了，检查一下 Wi-Fi 再试试？"
  "密码不对哦，忘了的话可以点这里重置"

坏例子：
  "Error 500: Internal Server Error"（用户看不懂）
  "操作失败，请重试"（没说为什么，没给出路）
```

### 空状态

**公式**: [当前状态说明] + [引导行动]

```
好例子：
  "还没有收藏内容，去发现页逛逛？"
  "你的购物车空空如也 🛒 去看看今日推荐"

坏例子：
  "暂无数据"（冷冰冰）
  "No items found"（不说人话）
```

## 文案打磨技巧

### 1. 删字大法

```
原文：我们的产品能够帮助用户非常高效地完成各种任务
打磨：帮你快速搞定事情
再磨：快速搞定
```

### 2. 具体化

```
原文：大量用户好评
打磨：23,847 人给了 5 星
```

### 3. 场景化

```
原文：支持离线使用
打磨：地铁里没信号也能用
```

### 4. 口语化

```
原文：该功能旨在优化用户的使用体验
打磨：用起来更顺手了
```

### 5. 对比法

```
原文：我们的速度很快
打磨：别人要 3 分钟，我们只要 3 秒
```

## 文案自检清单

写完后逐项检查：

- [ ] **目标明确**: 这段文案要用户做什么？一个目标，不是三个
- [ ] **用户视角**: 是"你能得到什么"，不是"我们有什么"
- [ ] **具体性**: 有数字、有场景、有细节吗？
- [ ] **口语化**: 说出来自然吗？朋友之间会这么说话吗？
- [ ] **无 AI 味**: 经过 De-AI-ify 处理了吗？
- [ ] **够短**: 能不能再删一半？
- [ ] **有行动**: 用户看完知道下一步做什么吗？
- [ ] **有情绪**: 能引发什么感受？（好奇/紧迫/安心/开心）

## 与 De-AI-ify + Humanizer 的配合

```
Step 1: 用本技能的模板和公式写初稿
  ↓
Step 2: De-AI-ify 去掉 AI 常用词和工整结构
  ↓
Step 3: Humanizer 加入口语化、情感、具体细节
  ↓
Step 4: 用本技能的自检清单做最终检查
  ↓
Step 5: 提交审核
```

---

## 限制

1. **不替代品牌策略** — 文案是品牌策略的执行层，不是策略本身
2. **不替代用户研究** — 洞察来自对用户的理解，不是凭空想象
3. **需要人工审核** — AI 生成的文案必须经过人工打磨和 De-AI-ify
