# Humanizer 技能

## 概述

**来源**：孟健老师推荐的实用 Skill

De-AI-ify 去掉了 AI 的常用词和工整结构，Humanizer 进一步让文本更像人写的：
1. **加入个性化细节**
2. **注入情感和态度**
3. **自然的停顿和节奏**
4. **真实的犹豫和思考**

---

## 核心理念

### 人写的文本 vs AI 生成的文本

| 特征 | AI 生成 | 人写的 |
|-----|---------|--------|
| 用词 | 重复、正式 | 多样、口语化 |
| 结构 | 工整、对称 | 不规则、有节奏 |
| 情感 | 中性、客观 | 有态度、有情感 |
| 思考 | 直线型 | 有犹豫、有转折 |
| 细节 | 抽象、泛泛 | 具体、生动 |

---

## 人性化的 8 个技巧

### 1. 加入思考过程

❌ **AI 味儿**：
```
这个方法很有效。
```

✅ **人性化**：
```
这个方法……怎么说呢，挺管用的。
至少我试过几次都没翻车。
```

### 2. 加入犹豫和不确定

❌ **AI 味儿**：
```
这是最好的解决方案。
```

✅ **人性化**：
```
这个方案应该是比较靠谱的吧。
至少我现在想不到更好的了。
```

### 3. 加入自嘲和幽默

❌ **AI 味儿**：
```
我完成了这个项目。
```

✅ **人性化**：
```
项目总算做完了，虽然中间差点翻车。
```

### 4. 加入真实的感受

❌ **AI 味儿**：
```
这个问题比较复杂。
```

✅ **人性化**：
```
这问题真的让我头疼了好几天。
```

### 5. 使用"我"的视角

❌ **AI 味儿**（客观陈述）：
```
这个方法可以帮助开发者提高效率。
```

✅ **人性化**（第一人称）：
```
我用这个方法，至少省了 3 个小时。
```

### 6. 加入对话感

❌ **AI 味儿**（单向陈述）：
```
这个技术有三个优点：1. xxx 2. xxx 3. xxx
```

✅ **人性化**（对话感）：
```
这个技术有啥好处呢？
首先……其次……最后……
你可能会问，那缺点呢？
```

### 7. 加入停顿和留白

❌ **AI 味儿**（连续输出）：
```
这个项目很成功，我们完成了所有目标，团队表现很好，未来我们会继续努力。
```

✅ **人性化**（停顿 + 留白）：
```
这个项目还算成功吧。

目标都完成了。

团队表现不错。

未来？先歇两天再说。
```

### 8. 加入具体的数字和细节

❌ **AI 味儿**（抽象）：
```
我花了很多时间研究这个问题。
```

✅ **人性化**（具体）：
```
我花了一整天（大概 8 个小时）研究这个问题。
```

## 实战示例

### 示例 1：技术文章开头

**AI 生成**：
```
今天分享一个技术话题。这个技术非常实用。
```

**De-AI-ify 后**：
```
今天聊个技术问题，挺实用的。
```

**Humanizer 后**：
```
今天聊个技术问题。

说实话，我之前也没想到这么有用。
直到上周踩了个坑，才发现这玩意儿是个宝。
```

---

### 示例 2：产品介绍

**AI 生成**：
```
我们的产品功能强大，性能稳定，用户体验好。
```

**De-AI-ify 后**：
```
我们产品功能够用，性能稳定，用起来挺顺手。
```

**Humanizer 后**：
```
我们产品……怎么说呢。

功能嘛，该有的都有，用起来挺顺手。
性能？至少我自己用了半年没翻过车。

当然，你要问完不完美？
那肯定不完美，但至少不会让你骂娘。
```

---

### 示例 3：经验分享

**AI 生成**：
```
通过这次项目，我学到了很多经验。希望分享给大家。
```

**De-AI-ify 后**：
```
这次项目学到点东西，写出来分享一下。
```

**Humanizer 后**：
```
这次项目……怎么说呢，踩的坑比走的路多。

但也算学到点东西吧。

写出来分享一下，希望你们别像我一样傻乎乎的踩。

（虽然我知道，该踩的坑你们还是会踩。但至少我尽力了。）
```

## 人性化的 5 个层次

### Level 1：基础口语化
```
这个方法挺好用的。
```

### Level 2：加入情感
```
这个方法挺好用的，我挺喜欢。
```

### Level 3：加入具体细节
```
这个方法挺好用的，至少帮我省了 3 个小时。
```

### Level 4：加入思考过程
```
这个方法……怎么说呢，挺好用的。
至少我试过几次都没翻车，应该算靠谱吧。
```

### Level 5：加入个性和幽默
```
这个方法……老实说，刚开始我是拒绝的。
觉得这玩意儿能有多靠谱？

结果试了一次，真香。

现在？离不开了。
```

## 不同风格的人性化

### 风格 1：轻松幽默

```
这个工具真的太好用了。

好用到什么程度呢？
好用到我用完之后，默默把之前的工具卸载了。

别问，问就是真香。
```

### 风格 2：真诚分享

```
这个工具我用了半年了。

说实话，刚开始也不太会用，踩了不少坑。
但慢慢地，发现它确实能解决我的问题。

现在基本上离不开了。
```

### 风格 3：专业严谨（仍然人性化）

```
这个工具在实际使用中表现不错。

我在 3 个项目中使用过，都达到了预期效果。
当然，也遇到过一些小问题，但都能解决。

总体来说，值得推荐。
```

## 与 De-AI-ify 的配合

**完整流程**：

```
第 1 步：AI 生成初稿
内容：这个方法非常有效，帮助我们解决了很多问题。

第 2 步：De-AI-ify（去掉 AI 常用词）
内容：这个方法挺管用的，帮我们解决了不少问题。

第 3 步：Humanizer（加入个性和情感）
内容：这个方法……怎么说呢，挺管用的。
至少我试过几次都没翻车。
```

## 检查清单

Humanizer 后，检查：

- [ ] 是否加入了思考过程（"怎么说呢"、"老实说"）？
- [ ] 是否加入了真实感受（"头疼"、"真香"）？
- [ ] 是否使用了第一人称（"我"、"我们"）？
- [ ] 是否有对话感（"你可能会问"）？
- [ ] 是否有停顿和留白（段落间隔）？
- [ ] 是否有具体数字和细节（"3 个小时"、"半年"）？
- [ ] 是否有个性和幽默（根据风格调整）？
- [ ] 是否仍然清晰易懂（不要过度人性化）？

## 常见误区

### 误区 1：过度人性化

❌ **过度**：
```
哎呀这个工具真的太太太太好用了！！！
我简直爱死它了！！！
你们一定一定一定要试试！！！
```

✅ **适度**：
```
这个工具挺好用的。
我用了半年，基本上离不开了。
推荐你们试试。
```

### 误区 2：失去专业性

❌ **不专业**：
```
这个技术嘛，反正就是那么回事儿，能用就行。
```

✅ **专业 + 人性化**：
```
这个技术在实际使用中表现不错。
我在 3 个项目中都达到了预期效果。
```

### 误区 3：只加停顿，不加内容

❌ **不够**：
```
这个方法挺好用的。

挺好用的。

真的挺好用的。
```

✅ **更好**：
```
这个方法挺好用的。

好用到什么程度呢？
至少帮我省了 3 个小时。
```
