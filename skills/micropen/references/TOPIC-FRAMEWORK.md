## 五种文章原型（v2.0 新增）

选题确定后，必须判断文章属于哪种原型。每种原型的写法重心不同，不要用通用模板硬套。

### 原型 1：调查实验型
**核心**：「我替你去做了这件事」
**写法重心**：过程叙事 + 层层递进的发现
**适用场景**：亲自下场做一件事，边做边报道发现
**骨架**：
```
遇到问题/好奇心 → 开始行动 → 踩坑过程 → 逐步发现 → 高潮结果 → 升华
```
**示例**：「我把 OpenClaw 的 53 个 Skill 全读了一遍」「我花了一天让 AI 从瞎子变火眼金睛」

### 原型 2：产品体验型
**核心**：「跟我一起玩」
**写法重心**：场景演示 + 真实感受 + 跟其他产品的自然对比
**适用场景**：拿到产品实际使用，带着读者一起体验
**骨架**：
```
引出产品 → 从基础功能开始（升番排列）→ 中间穿插真实感受 → 高潮案例 → 对比其他产品
```
**示例**：「CodeBuddy 和 Claude Code 我同时用了 30 天」

### 原型 3：现象解读型
**核心**：「你注意到了吗？背后是什么？」
**写法重心**：观察→好奇→研究→哲学升维
**适用场景**：观察到一个有趣的现象，层层深入分析
**骨架**：
```
现象描述 → 表面解释 → 更深的追问 → 找到真正原因 → 文化/哲学升维
```
**示例**：「Token 是这个时代的新货币」

### 原型 4：工具分享型
**核心**：「我发现了一个好东西」
**写法重心**：个人故事铺垫→工具展示→效果惊艳
**适用场景**：分享一个实用的工具/Skill，但必须用个人故事包裹
**骨架**：
```
个人困境/好奇心 → 发现这个工具 → 自己的使用过程 → 效果展示（要让人「卧槽」）
```
**示例**：「把维弈阁开源的那天晚上」

### 原型 5：方法论分享型
**核心**：「我把压箱底的东西掏给你了」
**写法重心**：每节有可执行行动 + 坦诚学习曲线 + 谦逊铺垫
**适用场景**：系统性分享自己积累的经验和方法论
**骨架**：
```
谦逊铺垫 → 方法论概述 → 逐条展开（每条有可执行行动）→ 坦诚学习成本 → 回扣+升华
```
**特殊要求**：
- 开头和结尾都要有谦逊铺垫，卸掉「我来教你」的傲慢感
- 每一节读完后读者手里有一个可以今天就执行的动作
- 必须坦诚说明学习曲线和常见失败点
**示例**：「想明白这 5 条，再来加 Agent」

## 选题输出模板

**每个选题必须填完下面这张卡才能进入创作阶段：**

```markdown
## 选题卡 #[编号]

### 基础信息
- **日期**：YYYY-MM-DD
- **领域**：AI 编程 / 工具建设 / 产品建设
- **独家度**：★★★★★（只有我能写 / 能写 / 谁都能写）
- **文章原型**：调查实验 / 产品体验 / 现象解读 / 工具分享 / 方法论（v2.0 新增）

### 备选标题（3 个，按热度排序）
1. 《[标题1]》— 热度 9/10
   - 公式：数字 + 成果 + 反差
2. 《[标题2]》— 热度 8/10
   - 公式：反直觉断言
3. 《[标题3]》— 热度 7/10
   - 公式：独家视角

### 热度评分（1-10）
- **时效性**（_/10）：48h 内热点？常青？
- **话题性**（_/10）：读者会转发吗？
- **独家性**（_/10）：只有我有的经验/数据？
- **情绪强度**（_/10）：能触发顿悟/共鸣/愤怒？
- **综合热度**：平均值

### 切入角度（4 选 1）
- [ ] **反直觉**：挑战读者既有认知
- [ ] **独家经验**：只有我踩过这个坑
- [ ] **技术解剖**：把一个黑盒拆开给你看
- [ ] **趋势判断**：提前看到一个变化

### 核心要点（3-5 个）
1. [论点 1]
2. [论点 2]
3. [论点 3]
4. [论点 4]（可选）
5. [论点 5]（可选）

### 预期情绪
- **主情绪**：顿悟 / 共鸣 / 愤怒 / 好奇 / 认同
- **读者反应**："原来如此" / "这跟我一样" / "没想到"

### 预期数据
- 阅读量：[预估]
- 收藏率：[预估]（> 5% 算优秀）
- 转发率：[预估]（> 3% 算优秀）

### 风险
- 会不会被觉得自夸？如何规避？
- 会不会被觉得太技术？如何通俗化？
- 有没有事实错误的风险？

### 所需素材
- [ ] 网络搜索：XXX、YYY
- [ ] 记忆检索：我之前做过 ZZZ 的经验
- [ ] 技术文档：AAA 的官方文档 / BBB 的源码
- [ ] 历史文章参考：[[文章1]]、[[文章2]]
```

## 热度评分标准

| 评分 | 含义 | 示例 |
|------|------|------|
| 10 | 独家暴击，必爆 | "我从 Copilot 源码里挖出了一个彩蛋" |
| 8-9 | 高热度 | "多 Agent 协作的真相" |
| 6-7 | 中热度 | "我用 Hermes 架构给自己团队做体检" |
| 4-5 | 中低热度 | "公众号发布工具对比" |
| 1-3 | 低热度 | "一些我最近在思考的问题" |

**低于 6 分的选题 → 直接毙掉，不写。**

## 切入角度的 4 个公式

### 公式 1：反直觉断言
```
「大多数人觉得 X，但其实 Y」
例：大多数人觉得多 Agent = 更智能。错。
```

### 公式 2：独家经验拆解
```
「我做了 X，踩了 Y 个坑，悟了 Z 个道理」
例：维弈阁从 12 到 13 个 Agent，我终于想明白了这件事
```

### 公式 3：技术黑盒解剖
```
「把 X 拆开，我发现 Y」
例：我用 Hermes 的 7 层架构，给自己的 AI 团队做了一次体检
```

### 公式 4：趋势预判
```
「X 还没火，但它会是 Y 的下一站」
例：Token 是这个时代的新货币（在大家还在谈模型参数的时候）
```

## 选题来源渠道（按优先级）

### Tier 1：独家来源（最高优先）
- 自己踩的坑（维弈阁 / microlab.top / Chromium 定制）
- 自己做的工具（MicroScope / MicroPub / MicroPen）
- 自己做的产品数据（microlab.top 访问量 / PawMBTI 用户反馈）

### Tier 2：二创 / 再阐释
- 读到的好文章（Hermes / gstack / Harness）
- 技术博客深度阅读后的再思考
- 开源项目源码分析

### Tier 3：热点跟进（谨慎使用）
- 推特热门 AI 工具
- HackerNews Top 10
- 小红书/知乎热榜

## 选题黑名单（不写）

- ❌ 纯教程类（"如何安装 XXX"）—— 除非有独家视角
- ❌ 产品评测类（"XXX 工具好不好用"）—— 没人关心
- ❌ 情绪宣泄类（"当前行业好卷"）—— 低端
- ❌ 鸡汤类（"AI 时代程序员该如何自处"）—— 重灾区
- ❌ 没有观点的"信息整理"—— 搬运工
- ❌ 超出主打领域的（金融/医疗/法律）—— 不懂不写

## 素材收集要求（Step 1）

### 必须串行，不并发
搜索引擎对高频并发请求敏感。**每次 web_search 间隔 2 秒以上**。

### 素材种类
1. **网络资讯**（时效性）
   - web_search：主题相关的最新文章/新闻
   - 来源：Twitter、公众号、HackerNews、GitHub trending

2. **技术文档**（权威性）
   - 官方 Docs、GitHub README、论文

3. **记忆库**（独家性）
   - `.weiyige/*/memory/` 下的个人经验
   - `content/wechat/*/` 下的历史文章
   - `MicroScope/projects/*/output/` 下的调研产出

4. **数据**（具体性）
   - GitHub Star 数、下载量、发版频率
   - 自己的项目数据（访问量、用户数）

### 素材卡模板

```markdown
## 素材卡 - [主题]

### 事实
- [事实 1]：[来源链接]
- [事实 2]：[来源链接]
- [事实 3]：[来源链接]

### 数据
- [数据 1]：[数值 + 来源]
- [数据 2]：[数值 + 来源]

### 引用
- [引用 1]：某人说过 "XXX"
- [引用 2]：某文章提到 "YYY"

### 个人经验
- 我在 [项目] 中遇到过 [场景]
- 我做过 [事情]，结果是 [结果]

### 技术细节
- [代码片段 / 配置 / 架构图]
```
