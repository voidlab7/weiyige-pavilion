## 作者已发布标题

### 现象级
```
《多 Agent 协作的真相：加 Agent 不难，难的是链路跑通》
  公式：反直觉断言 + 副标题解释
  热度：★★★★★
  为什么好：前半句立论点，后半句给真相

《我用 Hermes 的 7 层架构，给自己的 AI 团队做了一次体检》
  公式：独家视角 + 数字（7 层）+ 动词（体检）
  热度：★★★★★
  为什么好："体检"这个比喻精彩，给自己团队加人设

《从古法编程到 AI 搭子：一个公众号的诞生》
  公式：对比反差 + 独家视角
  热度：★★★★☆
  为什么好："古法编程"是作者发明的词，辨识度高
```

## 可以复用的标题公式

### 公式 1：数字 + 成果 + 反差
```
《从 12 到 13 个 Agent，我终于想明白了这件事》
《6 天 5 万 Star：Y Combinator 总裁的 AI 编程工作流》
《一天时间，一个 AI 从"瞎子"变"火眼金睛"》
```

### 公式 2：反直觉断言
```
《加 Agent 是最廉价的勤奋》
《大多数人都错了：AI 不是越多越强》
《Agent 不是工具，是员工》
```

### 公式 3：独家视角拆解
```
《我用 Hermes 的 7 层架构，给自己的 AI 团队做了一次体检》
《逆向 Copilot 那 3 天，我终于看懂了 AI 编程的本质》
《把 OpenClaw 拆开看：它比 Claude Code 强在哪？》
```

### 公式 4：趋势预判
```
《Token 是这个时代的新货币》
《Harness Engineering：AI 正式进入线束时代》
《2026 的 AI 产品，赢家都在造 Harness》
```

### 公式 5：场景 + 顿悟
```
《我让 Claude 写了一天代码，才发现它一直在骗我》
《凌晨三点，我对着 MiniMax 说："帮我识别这张图"》
《花了 10 块钱 Token，我才搞懂 Agent 怎么调度》
```

### 公式 6：对比冲击
```
《字流 vs MicroPub：公众号发布工具的一场内战》
《CodeBuddy 和 Claude Code，我同时用了 30 天》
《同一个任务，12 Agent vs 13 Agent 差多少？》
```

### 公式 7：行动召唤
```
《想明白这 5 条，再来加 Agent》
《搭多 Agent 系统之前，先搭这 3 个 Harness》
《想做 AI 编程？先装好这 7 个 Skill》
```

## 标题长度规律

- **最佳长度**：14-22 字
- **最长不超过**：28 字（手机显示会截断）
- **副标题**：可有可无，有则控制在 15 字内

---

## 标题 emoji 使用规则

### ❌ 不用
- 🔥🚀✨💡💯（AI 味，toxic）
- 😂🤣😭（过度情绪）

### ✅ 可用（克制）
- 一篇用 1 个，最多 2 个
- 作为"标点"使用，不作为"装饰"
- 位置：标题开头或核心词前

```
✅ ⚡ 多 Agent 协作的真相
✅ 🔬 我用 Hermes 架构给自己团队做了体检
⚠️ 实际上建议：**不用** emoji
```

## 标题禁用词

### 标题党套路（禁用）
- 震惊
- 重磅
- 必看
- 速看
- 火了
- 杀疯了
- 炸裂
- 绝了
- 封神
- 太牛了

### Low 词（禁用）
- 浅谈
- 初探
- 杂谈
- 漫谈
- 絮语
- 随想
- 一点思考
- 分享一下

### 标题党情绪词（禁用）
- 不要再 XXX 了
- 你以为的 XXX，其实是 YYY（过时了）
- 让 XXX 炸锅的 XXX

## 标题 A/B 测试建议

每次写完给出 3 个备选：
1. **保守版**：最稳，热度 7-8
2. **主推版**：最想用，热度 8-9
3. **激进版**：最搏出圈，热度可能爆也可能翻车

把 3 个标题放一起对比：
- 读者看到会先点哪个？
- 哪个更像作者会写的？
- 哪个符合平台推荐算法？（数字/反差/情绪）
