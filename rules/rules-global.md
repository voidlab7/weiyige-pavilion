# 维弈阁全局规则（Global Rules）

> 版本: v1.0 | 创建: 2026-04-16
> 借鉴: GSLB DevOps 的 G1-G13 全局规则独立文件设计
>
> **所有 12 个角色必须遵守。违反任何一条 = 产出无效。**

---

## W1: 配置优先

执行前读取 `.weiyige/project.yaml`（如存在）。字段缺失用默认值，文件不存在按全默认运行。

## W2: 角色不越界

每个角色只做 SOUL.md 中"主 owned"列出的事。不做"不做"列表中的事。具体边界见 PROTOCOL.md §1.2 RACI 矩阵。

## W3: 产出必须可验证

所有审查报告、PRD、设计文档等**必须写入文件**（不是只在对话中输出），且可通过 `read_file` 读取。声称"已完成"但无文件 = 未完成。

## W4: 门禁不可跳过

阶段流转必须通过对应 `gates/gate-*.md` 的逐条检查。`gates.strict_mode = true` 时，有未勾选项 = 阻止流转。默认模式下警告但允许继续。

## W5: 审核必须评分

审查类 Agent（锋/矩/绘/鉴/盾）完成审查后，**必须输出结构化评分**（见 `rules/review-scoring.md`），不可只给"通过/不通过"的结论。评分是门禁检查的质量评审部分。

## W6: 交接块强制

完成实质性工作后**必须输出交接块**（来源/状态/下游建议），格式见 PROTOCOL.md §2.2。没有交接块 = 工作未闭环。

## W7: 自检强制

输出交接块之前**必须执行完成前自检**（SOUL.md 中的「完成前自检」章节 + PROTOCOL.md §3.6 通用项）。有未通过项必须补充，不可忽略。

## W8: 记忆回写

完成实质性工作后检查是否有值得记录的教训/知识/偏好。有则写入对应 `memory/` 文件。关键工程约束同步写入 CodeBuddy Memory（`update_memory`）。

## W9: 计算型检查优先

矩（工程审查）和鉴（QA）的审查流程第一步**必须先执行计算型检查**（Lint/类型检查/测试），通过 `execute_command` 实际执行。禁止不执行就声称"检查通过"。

## W10: 串行流转

标准流程中，前一个阶段的 Agent 完成 + 门禁通过后，才能进入下一阶段。**禁止跳过阶段**。无依赖的审查（矩+绘）可并行，但都必须通过后才进入下一阶段。

## W11: Git 安全

- 涉及 `git push`、`rm -rf`、SCP 部署等**破坏性操作**不自动执行，提醒用户确认
- `git.protected_branches` 中的分支不自动 push
- 密钥/凭证禁止写入 git 跟踪的文件

## W12: Skill 调用真实性

所有在 SOUL.md/SKILLS.md 中声明的 Skill 流程步骤，如果需要工具调用（`execute_command`、`read_file`、`web_search` 等），**必须实际执行**，禁止声称"已执行"但实际跳过。

---

## 规则冲突解决

当规则之间冲突时，**取更严格的那个**。

优先级：W11（Git 安全）> W3（产出可验证）> W4（门禁）> 其他。
